# ワイヤーフレーム設計書

## 1. ホーム画面（app.html#home）

### 画面目的
ログイン後のメインページ。ユーザーの学習状況を一目で把握でき、次の学習アクションへすぐに遷移できるように設計。

### 各要素の説明

| 要素 | 説明 |
|------|------|
| ヘッダー | ロゴ＋プロフィールアイコン。プロフィール画像もしくはアバター表示 |
| ウェルカムメッセージ | 「こんにちは、[ユーザー名]さん」として個人化 |
| 継続ストリーク | 🔥アイコン＋「○日連続学習中」。15日以上で背景色変更（ゴールド） |
| 続きから学ぶ | 前回未完了の単元をカード表示。クリックで work 画面へ直遷移 |
| おすすめ単元 | AI が分析した弱点・学習進捗に基づく推奨。星印で難易度表示 |
| 今日の学習目標 | チェックボックス形式で、本日の進捗をリアルタイム反映 |

### インタラクション仕様

- **続きから学ぶカード**：タップで `app.html#work/[unitId]/[workType]` へ遷移
- **おすすめ単元カード**：タップで `app.html#unit/[unitId]` へ遷移
- **プロフィール**：タップで `app.html#menu` へ遷移
- **ストリーク**：15日以上でゴールド背景へ変更、20日以上で特別バッジ表示

### 問題タイプ別の入力インターフェース

#### タイプ1：choice（選択肢問題）
```
○ A. 選択肢1
○ B. 選択肢2
○ C. 選択肢3
○ D. 選択肢4
```
タップで選択、[答える] ボタンで確定。

#### タイプ2：sort（並び替え問題）
```
単語が複数のタグ状で表示される。
タップで順序をドラッグ＆ドロップ。

[I] [play] [soccer] [on] [Sundays]
      ↓ドラッグで並び替え
[I] [play] [soccer] [on] [Sundays]
```

#### タイプ3：transform（変形問題）
```
元の文：She is happy.
↓
形式：「過去形に変えよ」

入力欄：＿＿＿＿＿＿
       （キーボード入力）

キーボード入力で回答、[答える] で確定。
```

#### タイプ4：writing（英作文）
```
日本語：「彼はテニスをします」
↓
形式：「英語で書きなさい」

入力欄：＿＿＿＿＿＿＿＿＿＿
       （フルキーボード入力）

フリーテキスト入力。複数の正解を認識。
```

#### タイプ5：mistake（ミス発見）
```
文を提示：
「He play soccer on Sundays.」

形式：「この文の間違いを直しなさい」

複数選択肢またはテキスト入力で修正を提出。
```

### 各要素の説明

| 要素 | 説明 |
|------|------|
| 進捗インジケーター | 「○/○問」で現在位置を表示。プログレスバーで可視化 |
| 問題文 | 大きく表示し、下に日本語訳を小さく記載 |
| ヒント | 「ヒントを見る」で問題解き方のアドバイスを表示 |
| 選択肢 | 問題タイプに応じた入力形式（ラジオボタン、テキスト等） |
| フィードバック | 正解・不正解の理由を詳細に解説 |
| 関連例文 | その概念を深める別の例を提示 |
| 結果サマリー | 正答率、所要時間、単元全体の進捗を表示 |

### インタラクション仕様

- **問題画面**：選択肢をタップで選択、[答える] で確定
- **フィードバック画面**：自動表示、5秒後に [次へ] ボタンが活性化
- **ヒント**：「ヒントを見る」をタップで段階的に表示
- **結果画面**：各セクションをタップで詳細表示可能（アコーディオン）
- **スキップ**：問題をスキップして次へ進む（スキップ問題は後で復習）

## まとめ：ワイヤーフレーム設計の原則

### 1. ユーザー中心設計
- 中学生と保護者の両者にとって使いやすいインターフェース
- スマートフォン前提（390px幅）
- スクロール量を最小化

### 2. 段階的学習サポート
- 解説 → ワーク → テスト の3段階
- 即座のフィードバック
- 個別の質問対応

### 3. モチベーション維持
- 連続学習ストリーク
- 進捗の可視化
- 保護者からの励ましメッセージ

### 4. 一貫性
- 全画面でボトムナビゲーション表示
- 統一されたカード・ボタンデザイン
- 明確な色分け（完了=緑、未開始=グレーなど）

## 参考：レスポンシブ対応予定

現在のデザインは390px（iPhone SE規模）を基準としていますが、将来的に以下の対応を検討します：

- **幅768px以上**：タブレット対応（サイドバー表示など）
- **幅1024px以上**：デスクトップ対応（プレビューモード）

ただし、主力はスマートフォンアプリとします。
