## 中学校英語サポート＋

**作成日**: 2026年3月29日
**バージョン**: 1.0
**対象機能**: Q&A / 質問回答システム

## 1. 質問機能の位置付け

### 役割・価値
質問機能は、本サービスの**最大の差別化ポイント**です。

- **学習者にとって**: わからないことを即座に解決できる安心感
- **サービスにとって**: ユーザー定着率（リテンション）の向上、口コミによる獲得
- **事業にとって**: 人間による対応を武器とした競合との差別化

### 目指す状態
```
学習中の疑問
    ↓
「質問しよう」（心理的ハードルが低い）
    ↓
質問送信（匿名・簡単）
    ↓
24時間以内に親切な回答を受け取る
    ↓
理解が深まり、次のステップへ進める
    ↓
「またこのサービス使いたい」という好感度UP
```

## 2. 質問受付の考え方

### 2.1 対応範囲（IN SCOPE）

#### 教科書レベルの英語学習全般
- **文法**: 三単現のs、be動詞、過去形、不定詞など
- **単語**: 教科書掲載単語の意味、使い方
- **構文**: 「この文型はなぜこうなるのか」という構造的な質問

#### 学習方法に関する質問
- 「この単元のテスト対策のポイントは？」
- 「この問題がわからないときはどう考えればいい？」
- 「英語が苦手なのですが、どう勉強したらいい？」

#### サービス・教材に関する質問
- 「この問題の答えが合っているか確認してほしい」
- 「ここの解説がもっと詳しく知りたい」
- 「類似問題がもっとほしい」

#### 感情的サポート
- 「英語が嫌いになりかけています」
- 「テストが怖い」
（学習継続のモチベーション維持に重要）

### 2.2 非対応範囲（OUT OF SCOPE）

#### 宿題の代行
- ❌ 「この宿題の答えを教えてください」
- ⭕ 対応方法: 「わからないところはどこですか？」と質問を返し、わかるまでサポート

#### 中2以上の内容（当面）
- ❌ 「中2の現在完了について教えてください」
- ⭕ 対応方法: 「現在は中1の内容に特化しています。アップデート時にお知らせします」

#### 他教科（数学・国語など）
- ❌ 「数学の問題が解けません」
- ⭕ 対応方法: 「申し訳ございません。英語のみのサポートとなっております」

#### 個人情報を含む相談
- ❌ 「私の学校は〇〇中学で、〇〇先生の授業が...」
- ⭕ 対応方法: 「個人情報のため削除させていただきます。可能であれば学校名を除いて再度質問をお願いします」

#### 入試対策・受験相談
- ❌ 「高校入試に向けてアドバイスをください」
- ⭕ 対応方法: 「受験相談は提携の塾にお問い合わせください」

#### 医学的・心理的相談
- ❌ 「学習障害があるようですが」
- ⭕ 対応方法: 「医学的なご相談は医師にお願いします。学習面のサポートはいたします」

## 3. 質問フロー設計

### 3.1 質問送信フロー（ユーザー視点）

```
【質問画面】
    ↓
1. 質問前の注意文を表示
   （個人情報を含めない、30秒で読める）
    ↓
2. 質問タイプを6種類から選択
   （テンプレート化で質問しやすく）
    ↓
3. 関連する単元を選択
   （検索対象や文脈を明確化）
    ↓
4. 質問内容をテキストエリアに記入
   （最大500文字制限）
    ↓
5. 送信ボタンをタップ
    ↓
6. バリデーション
   ✓ 必須項目が入力されているか
   ✓ 個人情報が含まれていないか（AI検知）
   ✓ 適切な言葉遣いか（キーワード検知）
    ↓
7. 送信完了画面を表示
   「回答予定: 明日 18:00 まで」
    ↓
【クローズ】
```

### 3.2 回答処理フロー（スタッフ視点）

```
【質問管理ダッシュボード】
    ↓
1. 新規質問一覧を表示
   （受信順、優先度順）
    ↓
2. 質問を詳細表示
   - 質問タイプ、単元、内容
   - ユーザープロフィール（匿名）
    ↓
3. 自動FAQ検知
   「この質問は既知のFAQですか？」
   ⭕ YES → FAQへのリンクを自動返答
   ❌ NO → 手動回答に進む
    ↓
4. 回答を作成
   （テンプレートを参考に、カスタマイズ）
    ↓
5. 例文・画像などを添付
   （必要に応じて）
    ↓
6. 回答プレビュー
   「学習者の立場で、わかりやすいか」
    ↓
7. 送信
    ↓
8. ユーザーに通知
   「新しい回答が届きました」
    ↓
【クローズ】
```

### 3.3 フォローアップフロー

質問者が回答を読んだ後：

```
1. 「役に立ちましたか？」の5段階評価を表示
2. 点数に基づいて自動分類
   - 4〜5点: 満足
   - 2〜3点: 改善の余地あり → スタッフにフラグ
   - 1点: 不満足 → 再回答の検討

3. 追加質問を促す
   「まだわからないことはありますか？」
```

## 4. 質問テンプレート設計

### 4.1 テンプレート6種

質問画面で**ラジオボタン選択式**の6つのテンプレートを用意します。

#### テンプレート1: 解説が理解できない
```
【テンプレートテキスト】
「[単元名]の解説が理解できません。」

【ユースケース】
- 「三単現のs」のポイントカードの説明が漠然としている
- 「be動詞」の過去形がなぜそうなるのか理解できない

【例】
「三単現のsの解説で『1人の人やものについて話すとき』
と書いていますが、具体的にはどういうことですか？」

【回答方針】
別の言い方・図解・実例を複数提示して理解を促す
```

#### テンプレート2: 問題の答えがわからない
```
【テンプレートテキスト】
「[単元名]の[問題番号]の答えを教えてください。」

【ユースケース】
- ワークで特定の問題が解けない
- 選択肢の意味がわからない

【例】
「問題5で『He goes to school.』の『goes』が正解らしいのですが、
なぜ『go』ではなく『goes』なのか教えてください。」

【回答方針】
ヒントを段階的に提示し、最後に答えを示す
「考えてみるステップ」を設ける
```

#### テンプレート3: 文法・単語の使い方がわかりません
```
【テンプレートテキスト】
「[単語/文法]の使い方がわかりません。例文を教えてください。」

【ユースケース】
- 「does」と「do」の使い分け
- 「play」と「play with」の違い

【例】
「doとdoesの使い分けがわかりません。
例文を使って教えてくれると嬉しいです。」

【回答方針】
対比表を作成、複数の実例を提示
「覚え方」のコツを提示
```

#### テンプレート4: 解説を読んでもわかりません
```
【テンプレートテキスト】
「解説を読んでもわかりません。もっとやさしく教えてください。」

【ユースケース】
- 解説が難しく、用語が理解できない
- なぜそう考えるのかの前提がわからない

【例】
「『三単現のs』の規則について書いてありますが、
『ch/x/s/sh/z』で終わる動詞について説明が複雑で
理解できません。」

【回答方針】
用語をかみ砕く、段階を細かく分ける
中学1年生が理解できるレベルの言葉で書き直す
```

#### テンプレート5: テストに出やすいポイントは？
```
【テンプレートテキスト】
「テストに出やすいポイントはどこですか？」

【ユースケース】
- テスト直前に学習効率を上げたい
- 優先度をつけたい

【例】
「三単現のsの単元で、テストに出やすい部分は
どこですか？時間が限られているので...」

【回答方針】
「過去5年分の入試問題から、出題頻度が高いのは...」
という具体的な根拠を示す
```

#### テンプレート6: 答えが違う理由を教えてください
```
【テンプレートテキスト】
「[問題]の答えが[自分の答え]になったのですが、
なぜ間違いですか？」

【ユースケース】
- 自分の答えと正解を比較して、なぜダメなのかを知りたい
- 「ほぼ合っているのに」という場合

【例】
「問題3で『She go to school.』と書きましたが、
答えは『She goes to school.』です。
『She go』ではダメな理由をきちんと理解したいです。」

【回答方針】
間違えた理由を丁寧に解説（決して否定せず）
類似の引っ掛けポイントを提示
```

## 5. 回答テンプレート・トーン設計

### 5.1 基本トーン

すべての回答は以下のトーンで作成します：

| 項目 | 具体例 |
|------|--------|
| **褒める** | 「よい質問です！」「気づきましたね！」 |
| **励ます** | 「大丈夫です、多くの人が同じところで迷います」 |
| **わかりやすく** | 難しい用語を避ける、短い文 |
| **親切** | 「わかりにくければ聞いてください」 |

### 5.2 回答テンプレート5種

#### テンプレートA: 基本回答（最も一般的）
```
【構造】
1. 質問への理解を示す
2. 要点を1〜2文で簡潔に
3. 詳しい説明（図解・例文）
4. 確認や励まし

【例】
Q: 三単現のsの解説で『1人の人やものについて話すとき』
   と書いていますが、具体的にはどういうことですか？

A: よい質問ですね！簡単に言うと、「He」や「She」など、
   1人の人について説明するときに、動詞に「s」をつける
   という意味です。

   詳しく：
   ✓ I play soccer. （私は...） ← sなし
   ✓ He plays soccer. （彼は...） ← s付き
   ✓ She plays soccer. （彼女は...） ← s付き
   ✗ They play soccer. （彼ら） ← sなし

   「He や She = 1人」として覚えると簡単です！
   わかりましたか？
```

#### テンプレートB: 例文追加型
```
【構造】
1. 短い説明
2. 「例えば」と複数の例文を提示
3. 対比で違いを明示
4. 覚え方のコツ

【例】
Q: doとdoesの使い分けがわかりません。

A: 「do」と「does」の違いは簡単です。

   例1: I do my homework. （私は...）⟵ do を使う
   例2: She does her homework. （彼女は...）⟵ does を使う
   例3: They do their homework. （彼ら...）⟵ do を使う

   覚え方：「He/She/It」の場合だけ「does」。
   それ以外は「do」と覚えましょう！
```

#### テンプレートC: FAQ自動誘導型
```
【構造】
1. 簡潔な回答
2. より詳しい説明へのリンク
3. 次のステップの提案

【例】
Q: sとesの違いは？

A: この質問の詳しい解説はここに書いています：
   📖 【FAQ】sとes、どちらをつける？規則を完全解説

   上のリンクで、より詳しい説明と練習問題を見られます。
   ぜひ確認してみてください！
```

#### テンプレートD: 再質問促し型
```
【構造】
1. 部分的な回答
2. 相手の理解度を探る質問
3. より具体的な情報を得る工夫

【例】
Q: 三単現のsがわかりません。

A: 「わかりません」ということですが、
   具体的にはどの部分がわかりにくいですか？

   ☐ sをつけるルール（いつつける？）
   ☐ sをつけた形の読み方・発音
   ☐ 問題の中で「s」と「es」どちらにするか判断できない

   上から選んでいただくか、「その他」があれば
   教えていただけますか？
   より的確なサポートができます！
```

#### テンプレートE: 範囲外への対応
```
【構造】
1. 丁寧な説明
2. サービス範囲内で対応できることを提示
3. 次のステップの提案

【例】
Q: 高校入試に向けてアドバイスをください。

A: ご質問ありがとうございます。
   申し訳ないのですが、入試対策のご相談は
   このサービスの範囲外です。

   ただし、中1の英文法の基礎を完全に理解することが
   入試対策の最初のステップになります。

   もし「三単現のs」など基礎的な内容に
   質問があればお手伝いします！
```

### 5.3 回答作成時のチェックリスト

各回答を作成した後、以下をチェック：

- [ ] 敬語は正しいか（です・ます調）
- [ ] 「否定」ではなく「肯定的な言い方」になっているか
- [ ] 中学1年生が理解できる言葉を使っているか
- [ ] 例文は3個以上か（1個だけでは不十分）
- [ ] 「わかりましたか？」など相手を意識した言葉があるか
- [ ] 句読点は多すぎないか（読みやすいか）
- [ ] 絵文字や図解が活用されているか
- [ ] 最後に励ましや次のステップを示しているか

## 6. SLA（Service Level Agreement）

### 6.1 返信時間の目標

| 曜日 | 受付時間帯 | 目標返信時間 | 定義 |
|------|-----------|-----------|------|
| **平日** | 24時間 | 24時間以内 | 質問受付から回答送信まで |
| **土日祝** | 24時間 | 翌平日18時以内 | 金曜夜の質問は月曜まで |
| **夜間質問** | 22:00以降 | 翌朝10:00以内 | 夜間受付は翌朝対応 |

### 6.2 返信時間の詳細ルール

```
【平日の例】
月曜 9:00 質問受付
  → 火曜 9:00 まで に回答を目標

【土日祝の例】
金曜 19:00 質問受付
  → 月曜 18:00 まで に回答を目標

【夜間質問の例】
平日 23:00 質問受付
  → 翌日 10:00 までに回答を目標
```

### 6.3 SLA達成率の目標

| KPI | 目標値 | 測定方法 |
|-----|--------|---------|
| **返信率** | 98%以上 | 受付質問に対し、回答または範囲外通知を送信した率 |
| **SLA達成率** | 95%以上 | 目標返信時間内に回答した率 |
| **満足度** | 4.0 / 5.0以上 | 利用後アンケートの平均スコア |
| **再質問率** | 10%以下 | 回答後に同じ内容で再質問した割合 |

## 7. FAQ化方針

### 7.1 FAQ化のトリガー

```
同じ質問が3件以上受信
    ↓
スタッフが FAQ 記事として作成
    ↓
質問管理ダッシュボードに登録
    ↓
その後の同一質問は自動的に FAQ へのリンク返答
```

### 7.2 FAQ化する質問の例

| # | よくある質問 | 想定回答項目 |
|---|------------|-----------|
| 1 | 「sとesの違いは？」 | 規則表、発音、練習問題 |
| 2 | 「doesってどこで使う？」 | do/does の使い分け、例文 |
| 3 | 「三単現のsが必要な理由は？」 | 英文法の歴史背景、納得できる説明 |
| 4 | 「問題が間違った理由がわかりません」 | よくある誤り5パターン |
| 5 | 「テスト対策で何を優先すればいい？」 | 優先順位、学習計画 |

### 7.3 FAQ の管理・更新

- **新規登録**: 月1回の定例会で確認・承認
- **レビュー**: 3ヶ月ごとに内容が古くなっていないか確認
- **廃止**: ユーザーニーズが変わった場合、アーカイブ化

## 8. 個人情報・安全管理

### 8.1 個人情報保護ポリシー（質問機能）

#### 禁止事項（ユーザーに事前告知）
- 本名・学校名・住所などの個人情報入力
- 友達の名前、先生の名前
- 学校行事の詳細情報
- 親の職業・年収などプライベート情報

#### 検出方法
1. **自動検知**: キーワードフィルタ（学校名、地名など）
2. **手動確認**: 送信前にスタッフが簡易チェック
3. **事後対応**: 個人情報が含まれていた場合、削除して対応

#### 回答時の注意
- 「〇〇中学の田中さん」ではなく「田中さん」と呼ぶ
- 学校固有の事情には触れない
- 回答内容を他のユーザーに見せないよう留意

### 8.2 モデレーション基準

#### 自動削除対象
- [ ] 暴力的・差別的な言葉
- [ ] スパム・広告リンク
- [ ] わいせつな内容
- [ ] 他ユーザーへの誹謗中傷

#### 手動確認が必要
- [ ] いじめの相談（学校カウンセラーへの誘導）
- [ ] 自傷・自殺関連（相談窓口へのリンク）
- [ ] 依存症的な学習（精神的負荷への配慮）

### 8.3 情報の扱い（スタッフ向け）

- **機密情報**: 個人を特定する情報は一切外部に漏らさない
- **バックアップ**: 質問・回答データは暗号化で保管
- **アクセス制御**: スタッフ全員が同じダッシュボードを見えないようにする（担当質問のみ表示）
- **保管期限**: ユーザー削除時にすべての質問・回答記録も削除

## 9. 破綻させない内部ルール

### 9.1 キャパシティ管理

#### 1日の対応上限
```
初期段階（利用者500名以下）
├─ 予想質問数: 10〜30件/日
├─ スタッフ人数: 1名（他業務兼任）
└─ 対応上限: 20件/日
   （超過分は翌日対応）

拡大段階（利用者5,000名）
├─ 予想質問数: 50〜100件/日
├─ スタッフ人数: 2名（専任）
└─ 対応上限: 50件/日
   （超過分は翌日以降対応）
```

#### 同一ユーザーの質問上限
- **1日あたり最大3件**
- 4件目以降は「たくさん質問いただきありがとうございます。
  後ほど改めて対応させます」と表示

#### 理由
- スタッフのバーンアウト防止
- ユーザーによる過度な依存防止
- サービス品質維持

### 9.2 質問パターン別の効率化

#### 頻出質問への対応
- FAQ へ自動リダイレクト（スタッフの手作業削減）
- AI による自動回答（試験運用：6ヶ月目以降）

#### テンプレート回答の活用
- 同じ構造の質問には、テンプレートをカスタマイズして対応
- 100%テンプレートではなく、個別要素を加える（温かみ維持）

#### スタッフスキルの段階化
```
レベル1（新人）: テンプレート回答のみ
  - 「規則の説明」「簡単な例文提示」

レベル2（中堅）: テンプレート応用
  - より複雑な質問に対応
  - テンプレートをカスタマイズ

レベル3（ベテラン）: 全対応
  - 困難な質問、グレーゾーンの判断
  - 新人研修・FAQ作成
```

### 9.3 スタッフの負担軽減

#### 時間短縮の工夫
- 一括検索機能（同じ質問を過去から検索）
- 「コピペOK」な回答テンプレート
- 短縮形の定義（「三単」「被験者」→「be動詞」など）

#### メンタルヘルス
- 1日の対応件数上限の厳守
- 週1回のミーティングで悩みを共有
- 月1回のスキルアップ研修

## 12. KPI・監視項目

### 12.1 定期レポート（月次）

| KPI | 計測方法 | 目標値 | 閾値|
|-----|---------|--------|------|
| **受信質問数** | ダッシュボード | 前月の 120% | 150件以上で警告 |

| **平均返信時間** | (送信日時 - 受信日時) の平均 | 12時間 | 24時間超で要改善 |
| **SLA達成率** | (SLA内回答数 / 全回答数) × 100% | 95% | 90%未満で要改善 |

| **再質問率** | (再質問数 / 回答数) × 100% | 10%以下 | 15%超で回答品質を検討 |
| **FAQ化件数** | 月次登録数 | 2件以上 | 目標達成でインセンティブ |

## 13. トラブルシューティング

### 13.1 よくあるトラブルと対応

| トラブル | 原因 | 対応 |
|---------|------|------|
| **SLA破綻** | スタッフ不足 / 質問激増 | 外部ライターの雇用、AI自動回答導入 |
| **満足度低下** | 回答品質の低下 | 研修強化、テンプレート見直し |
| **同じ質問が多い** | ユーザーニーズに解説がマッチしていない | 該当する解説の改善・強化 |
| **個人情報漏出** | スタッフのモラルハザード | 契約更新時に再教育、アクセスログ監視 |
| **過度な依存ユーザー** | 1日3件以上の質問常連 | 「自習の時間も大切」という促し、利用制限 |

### 13.2 エスカレーション

**対応できない質問が来た場合の流れ：**

```
スタッフが判断できない質問
    ↓
スーパーバイザーに相談
    ↓
方針を決定（対応 / 非対応 / 部分対応）
    ↓
マニュアルに追記
    ↓
次回以降は同じ判断を適用
```

## 付録: テンプレート回答の具体例

### 例1: テンプレートA（基本回答）の完成版

**Q**: 三単現のsをいつ使うのか、使わないのか、規則がよくわかりません。

**A**:
```
よい質問ですね！三単現のsは、「ルール」と「例外」の2つを
理解すると、すっきり解決します。

【ルール】
He や She など、「1人の人・1個のもの」について
現在のことを言うときに、動詞に「s」をつけます。

【例】
✓ She plays soccer. （彼女は...） ← Sheなので「s」
✓ He studies English. （彼は...） ← Heなので「s」
✓ The cat sleeps. （その猫は...） ← 「1個のもの」なので「s」
✗ I play soccer. （私は...） ← Iなので「s」なし
✗ You study English. （あなたは...） ← Youなので「s」なし
✗ They sleep. （彼らは...） ← 複数なので「s」なし

【覚え方のコツ】
「He/She/It」の3つの場合だけ「s」をつけると
覚えると簡単です！

どうですか？イメージがつきましたか？
```

### 例2: テンプレートB（例文追加型）の完成版

**Q**: does と do の使い分けが全くわかりません。

**A**:
```
わかりやすく説明しますね！

【簡潔な答え】
「do」と「does」は「〜しますか？」という疑問文を
作るときに使い分けます。

【パターン別の例】
1) I / You / We / They の場合 → do を使う
   Do you like English?
   （あなたは英語が好きですか？）

2) He / She / It の場合 → does を使う
   Does she like English?
   （彼女は英語が好きですか？）

【覚え方のコツ】
He/She/It の場合だけ「does」。
それ以外は全部「do」！

この規則は「三単現のs」と同じです。
確認問題をいくつか解いてみてください。
```
