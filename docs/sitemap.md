# サイトマップ・画面遷移設計

## 概要

中学校英語サポート＋は、スマートフォン向けのシングルページアプリケーション（SPA）として設計されています。ハッシュベースのルーティングを採用し、ユーザーの学習状況に応じた流動的な画面遷移を実現します。

---

## 画面一覧（10画面）

### 1. LP（index.html）
**目的**：未ログインユーザーへの集客ページ

- サービス説明（キャッチコピー、提供価値）
- 他サービスとの比較
- 保護者の声（モニター評価）
- 料金プラン表示
- 無料体験・会員登録への導線

**ターゲット**：初めてのユーザー、保護者

---

### 2. ホーム（app.html または app.html#home）
**目的**：ログイン後のトップ画面

**主要セクション**：
- ユーザー挨拶（「こんにちは、○○さん」）
- 学習継続ストリーク（「○日連続学習中」バッジ）
- 「続きから学ぶ」（最後に学習した単元へのショートカット）
- 「おすすめ単元」（AIによるパーソナライズ化）
- ボトムナビゲーション

**ターゲット**：ログイン済みユーザー

---

### 3. 単元一覧（app.html#units）
**目的**：中学1年で学ぶすべての英文法単元を表示

**表示形式**：
- 単元カード形式（各カード：タイトル、難易度、推定時間、進捗バー）
- カテゴリ分け（例：品詞、時制、疑問詞など）
- フィルター機能（難易度別、進捗別）
- 検索機能

**ターゲット**：学習ユーザー

---

### 4. 単元詳細（app.html#unit/:unitId）
**目的**：特定の単元の学習内容を階層的に提示

**3つのタブ**：

#### タブ1：解説（explanation）
- 文法ルール説明（テキスト＋図解）
- 重要ポイント（黄色ハイライト）
- 用途別例文（肯定文、否定文、疑問文など）
- よくある間違い（アコーディオン）

#### タブ2：ワーク（work）
- 「基礎を固める」（選択肢＋並び替え）
- 「応用に挑戦」（変形＋英作文）
- ボタン：各問題セットへの遷移

#### タブ3：テスト（test）
- 「小テストを受ける」（10問、制限時間15分）
- テスト前注意（再受験可、複数回受けて最高点が記録される）
- ボタン：テスト開始

**ターゲット**：学習ユーザー

---

### 5. ワーク画面（app.html#work/:unitId/:workType）
**目的**：問題演習を実施、即座のフィードバック提供

**構成**：
- 進捗インジケーター（「○/○問」）
- 問題文表示
- 入力欄またはボタン（問題タイプによる）
- フィードバック表示（正解・不正解時のコメント）
- 「次へ」「終了」ボタン

**workType の種類**：
- `basic-choice`：単元詳細から「基礎を固める」→選択肢問題
- `basic-sort`：単元詳細から「基礎を固める」→並び替え問題
- `advanced-transform`：単元詳細から「応用に挑戦」→変形問題
- `advanced-writing`：単元詳細から「応用に挑戦」→英作文
- `test`：小テスト（10問セット）

**ターゲット**：学習ユーザー

---

### 6. 質問画面（app.html#question）
**目的**：わかりやすい形式で質問を受け付ける

**構成**：
- テンプレート選択（「この部分がわかりません」「例文の使い分けがわかりません」など）
- テキスト入力欄（質問内容、最大500文字）
- 参考画像アップロード（オプション）
- 「質問を送信」ボタン
- 送信後確認画面（「○時間以内に回答します」のメッセージ）

**ターゲット**：質問がある学習ユーザー

---

### 7. 質問履歴（app.html#question-history）
**目的**：過去の質問と回答を管理・復習

**表示形式**：
- タイムライン形式（最新から順）
- 各質問カード：質問内容、質問日時、回答ステータス（回答済み/保留中）
- 各回答カード：専門家の回答（展開可能）
- 回答の評価機能（役に立った/役に立たなかった）

**ターゲット**：質問を送信したユーザー

---

### 8. 学習記録（app.html#records）
**目的**：学習進度を可視化、モチベーション維持

**セクション構成**：

#### セクション1：学習習慣
- 連続日数カウンター（「○日連続」）
- 週間カレンダー（7日間、学習した日をチェック表示）
- 目標設定・編集（月間学習日数目標など）

#### セクション2：単元別進捗
- 進捗バー（全単元の完了率）
- 各単元カード（完了ステータス、得点率）
- 得意な単元・苦手な単元の表示

#### セクション3：正答率分析
- 全体の正答率グラフ（週単位）
- 問題タイプ別の正答率（選択肢、書き換え、英作文など）
- トレンド分析（向上している分野の表示）

**ターゲット**：学習ユーザー、保護者

---

### 9. 保護者確認画面（app.html#parent）
**目的**：保護者が子どもの学習状況を確認

**表示情報**：
- 今週のまとめ（学習日数、問題数、平均得点）
- 単元別進捗一覧（完了・進行中・未開始の分類）
- 学習時間トレンド（グラフ表示、直近4週間）
- 質問内容サマリー（何について質問が多いか）
- コメント機能（保護者から子どもへのメッセージ送信）

**ターゲット**：保護者、学習管理者

---

### 10. メニュー（app.html#menu）
**目的**：設定・その他機能への導線

**メニュー項目**：
- アカウント設定（プロフィール、メールアドレス変更）
- プラン変更（料金プラン変更ページへのリンク）
- 保護者画面（app.html#parent へのリンク）
- 質問履歴（app.html#question-history へのリンク）
- ヘルプ・FAQ
- お問い合わせ
- ログアウト

**ターゲット**：ログイン済みユーザー

## 画面遷移フロー

### フロー1：初回ユーザー
```
index.html（LP）
├─→ [無料体験ボタン] → app.html#home（ログイン後）
```

### フロー2：学習を始める
```
app.html#home（ホーム）
├─→ [続きから学ぶ] → app.html#work/[unitId]/[workType]（直接問題へ）
│
app.html#home（ホーム）
├─→ [おすすめ単元カード] → app.html#unit/[unitId]（単元詳細へ）
```

### フロー3：単元から学習開始
```
app.html#unit/s01（単元詳細）
├─→ [「ワークを始める」タブ内ボタン] → app.html#work/s01/basic-choice（基礎問題）
│
app.html#unit/s01（単元詳細）
├─→ [「小テストへ」タブ内ボタン] → app.html#work/s01/test（テスト）
│
app.html#unit/s01（単元詳細）
├─→ [「質問する」ボタン] → app.html#question（質問画面）
```

### フロー4：ワーク完了後
```
app.html#work/s01/basic-choice（ワーク結果画面）
├─→ [「単元に戻る」] → app.html#unit/s01（単元詳細へ）
│
app.html#work/s01/basic-choice（ワーク結果画面）
├─→ [「ホームへ」] → app.html#home（ホームへ）
│
app.html#work/s01/basic-choice（ワーク結果画面）
├─→ [「応用に挑戦」] → app.html#work/s01/advanced-transform（応用問題へ）
```

### フロー5：ボトムナビゲーション
```
【全ページから遷移可能】
ボトムナビゲーション
├── [ホーム] → app.html#home
├── [まなぶ] → app.html#units
├── [きろく] → app.html#records
└── [メニュー] → app.html#menu
        ↓
    メニュー内から
    ├── [保護者画面] → app.html#parent
    └── [ログアウト] → index.html（LP）
```

### フロー6：質問関連
```
app.html#question（質問画面）
├─→ [送信] → 確認画面 → app.html#home
│
app.html#question-history（質問履歴）
├─→ [質問をタップ] → 詳細表示・回答確認
```

## スマートフォンレイアウト考慮事項

### ディスプレイ幅
- ターゲット幅：375px～425px（iPhone SE～iPhone 15）
- デザイン基準：390px

### レイアウト構造
```
┌──────────────────────┐  高さ：可変
│     ヘッダー         │
├──────────────────────┤  高さ：60px
│                      │
│     コンテンツ       │  高さ：可変（スクロール可）
│                      │
├──────────────────────┤  高さ：64px
│  ボトムナビゲーション│  固定配置
└──────────────────────┘
```

### スクロール動作
- ボトムナビゲーション：常に固定配置（画面下部）
- ヘッダー：スクロール時に隠れる可能性有（UXレビュー時に検討）
- コンテンツ：スクロール可能

## 状態遷移図（ユーザーフロー）

```
【未ログイン】
    ↓
index.html（LP）
    ↓
[無料体験 または 会員登録]
    ↓
【ログイン】
    ↓
app.html#home（ホーム）
    ↓
[学習選択]
    ├→ app.html#units → app.html#unit/[unitId] → [学習タイプ選択]
    └→ app.html#home → [おすすめ単元] → app.html#unit/[unitId]
    ↓
[学習実施]
    ├→ app.html#work/[unitId]/[workType]（問題解く）
    ├→ 結果表示・フィードバック
    └→ app.html#unit/[unitId] or app.html#home へ戻る
    ↓
[復習・記録確認]
    ├→ app.html#question-history（質問確認）
    └→ app.html#records（学習記録確認）
    ↓
【継続学習 or ログアウト】
```

## API連携予定

### フロントエンド→バックエンド通信

| 機能 | HTTP メソッド | エンドポイント例 | 役割 |
|------|------------|----------|------|
| ホーム取得 | GET | `/api/v1/home` | ユーザー情報、継続日数、おすすめ単元 |
| 単元一覧 | GET | `/api/v1/units` | すべての単元データ |
| 単元詳細 | GET | `/api/v1/units/{unitId}` | 解説、問題セット |
| ワーク実施 | POST | `/api/v1/work/{workId}/submit` | 問題の回答送信、採点 |
| 質問送信 | POST | `/api/v1/questions` | 質問データ保存 |
| 質問履歴 | GET | `/api/v1/questions` | ユーザーの質問・回答一覧 |
| 学習記録 | GET | `/api/v1/records` | 学習統計、進捗データ |
| 保護者情報 | GET | `/api/v1/parent/summary` | 親向けサマリー |

- 10画面実装

## 参考：ページロード性能目標

| ページ | 初回読み込み | 2回目以降 | 目標 |
|-------|-----------|---------|------|
| LP | 3秒以下 | 1秒以下 | Google PageSpeed 70点以上 |
| app.html | 2秒以下 | 0.5秒以下 | ジャンク無し、60fps |
| ワーク画面 | 1秒以下 | 0.3秒以下 | 即レスポンス |

キャッシング戦略：
- Service Worker による効果的なキャッシング
- 問題セットは事前キャッシュ（初回DL時に30問分）
- APIレスポンスは30秒キャッシュ
