## 中学校英語サポート＋

**作成日**: 2026年3月29日
**バージョン**: 1.0
**対象デバイス**: スマートフォン（iOS / Android）

---

## 1. デザイン哲学・ガイドライン

### デザイン原則

1. **シンプルさ**
   - 中学1年生が直感的に操作できることを最優先
   - 機能を絞り、各画面の役割を明確にする
   - 余白を活用して視認性を確保

2. **信頼感**
   - 堅実で落ち着いた配色
   - 教員監修マークの配置で信頼性を演出
   - エラーメッセージは親切で励ましの言葉を含む

3. **親近感**
   - ユーザーの名前を呼んでパーソナライズ
   - 学習継続を褒める表現（「3日連続！」など）
   - アイコンは親しみやすいデザイン

4. **導線の明確さ**
   - 次に何をしたらいいかが常に示唆されている
   - 色・サイズ・位置で優先度を示す
   - 戻るボタンは常に配置

## 2. ビジュアルアイデンティティ

### カラーパレット

#### Primary Color（メインカラー）
- **色**: #2563EB（Blue）
- **用途**: ボタン、見出し、重要な要素
- **RGB**: rgb(37, 99, 235)
- **意味**: 「信頼」「安定」を表現

#### Secondary Color（サポートカラー）
- **色**: #10B981（Green）
- **用途**: 正解、完了、成功時のフィードバック
- **RGB**: rgb(16, 185, 129)
- **意味**: 「成長」「達成」を表現

#### Accent Color（アクセントカラー）
- **色**: #F59E0B（Amber）
- **用途**: 連続学習バッジ、重要な通知
- **RGB**: rgb(245, 158, 11)
- **意味**: 「活力」「注意」を引く

#### Background Color
- **色**: #F8FAFC（Light Slate）
- **用途**: ページ背景
- **RGB**: rgb(248, 250, 252)

#### Surface Color
- **色**: #FFFFFF（White）
- **用途**: カード、入力フィール
- **RGB**: rgb(255, 255, 255)

#### Text Primary
- **色**: #1E293B（Dark Slate）
- **用途**: 本文、見出し
- **RGB**: rgb(30, 41, 59)

#### Text Secondary
- **色**: #64748B（Medium Slate）
- **用途**: 補足テキスト、説明
- **RGB**: rgb(100, 116, 139)

#### Error Color
- **色**: #EF4444（Red）
- **用途**: 不正解、エラーメッセージ
- **RGB**: rgb(239, 68, 68)

#### Border Color
- **色**: #E2E8F0（Light Border）
- **用途**: 区切り線、分割線
- **RGB**: rgb(226, 232, 240)

### タイポグラフィ

#### フォント選定
- **日本語**: Noto Sans JP
- **英数字**: SF Pro Display / -apple-system（iOS）、Roboto（Android）
- **理由**: 読みやすさ、収録文字数の多さ、デバイス標準フォントの活用

#### フォントサイズ・ウェイト設定

| 要素 | サイズ | ウェイト | 行高 | 用途 |
|------|--------|--------|------|------|
| **H1（見出し1）** | 24px | 700 Bold | 32px | ページタイトル |
| **H2（見出し2）** | 20px | 700 Bold | 28px | セクションタイトル |
| **H3（見出し3）** | 16px | 600 SemiBold | 24px | カードタイトル |
| **本文** | 15px | 400 Regular | 24px | 解説文、説明テキスト |
| **小テキスト** | 13px | 400 Regular | 20px | 補足、メタ情報 |
| **ボタンテキスト** | 16px | 600 SemiBold | 24px | ボタン内テキスト |
| **キャプション** | 12px | 400 Regular | 18px | 最小限の補足情報 |

#### 行間の考慮
- 1行の文字数が55〜75文字を想定した行高設定
- 特に「解説」や「質問回答」は読みやすさ重視で行高を広め（24px以上）に設定

### スペーシング規則（8pxグリッドシステム）

| サイズ | ピクセル値 | 用途 |
|--------|----------|------|
| **xs** | 4px | 最小の間隔（テキスト内など） |
| **sm** | 8px | 要素内の小さな余白 |
| **md** | 12px | カード内の要素間隔 |
| **lg** | 16px | セクション内の主要な間隔 |
| **xl** | 24px | セクション間の間隔 |
| **2xl** | 32px | ページ主要セクション間 |

#### ページレイアウト単位

| 用途 | 値 |
|------|-----|
| **ページ左右余白** | 16px |
| **カード・セクション最大幅** | 100%（スマホ表示固定） |
| **カード角丸** | 16px |
| **ボタン角丸** | 12px |

## 8. アクセシビリティ設計

### WCAG 2.1 AAA 準拠の取り組み

#### 色覚障害への対応
- 色情報のみに頼らない（形・テキストラベルを併用）
- コントラスト比: テキスト 4.5:1 以上（AAA水準）
- 例：正解：✅ + 「正解」テキスト + 緑色

#### テキスト読みやすさ
- 最小フォントサイズ: 13px（推奨15px）
- 行高: 本文24px以上
- 1行あたり文字数: 55〜75文字

#### キーボード操作
- すべてのボタンにフォーカス状態を表示
- フォーカスオーダーは視覚的順序に準拠
- Tab キーで操作可能

#### スクリーンリーダー対応
- img タグに alt 属性を必ず設定
- ボタンに aria-label を設定
- フォーム要素に label と関連付け

## 9. パフォーマンス設計

### ページロード最適化
- **目標FCP（First Contentful Paint）**: 1秒以下
- **目標LCP（Largest Contentful Paint）**: 2.5秒以下
- **目標CLS（Cumulative Layout Shift）**: 0.1以下

### 画像最適化
- WebP 形式での提供（フォールバック: PNG/JPG）
- 最大解像度: デバイスピクセル比 2倍（2x）まで
- 遅延読み込み: スクロール範囲外の画像は読み込み遅延

### キャッシング戦略
- 静的資産（CSS、JS）: 1年キャッシュ
- HTML: キャッシュしない（常に最新取得）
- API レスポンス: 5分〜1時間（エンドポイント別）
