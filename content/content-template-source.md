## 問題タイプの定義

### 1. choice（選択肢問題）
```json
{
  "id": "qXX-q01",
  "type": "choice",
  "question": "問題文（日本語と英語の混在可）",
  "choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  "answer": 1,                      // 正解インデックス（0から始まる）
  "explanation": "解説",
  "hint": "オプション：ヒント"
}
```
- 用途：文法判断、意味理解、穴埋め選択
- 選択肢数：4個（レスポンシブに対応可能）
- 難易度調整：ひっかけ選択肢の難度を調整

### 2. transform（変形問題）
```json
{
  "id": "qXX-q02",
  "type": "transform",
  "question": "変形を指示する文",
  "answer": "正解（文字列）",
  "hint": "ヒント",
  "explanation": "解説（変化ルール説明）"
}
```
- 用途：動詞の活用、単語の形変化
- 答え形式：テキスト完全一致（大文字小文字区別）
- 複数答え対応可能

### 3. sort（並び替え問題）
```json
{
  "id": "qXX-q03",
  "type": "sort",
  "question": "並び替えの指示",
  "words": ["単語1", "単語2", "単語3", "単語4"],
  "answer": "Word1 Word2 Word3.",
  "hint": "ヒント",
  "explanation": "解説"
}
```
- 用途：語順・文法構造の理解
- 単語数：通常4～6個
- 答え形式：スペース区切り、句点含む

### 4. mistake（誤り訂正問題）
```json
{
  "id": "qXX-q04",
  "type": "mistake",
  "question": "間違いを含む英文",
  "wrongWord": "誤った単語",
  "answer": "正しい形",
  "explanation": "訂正理由と解説"
}
```
- 用途：よくあるミスの防止、理解度確認
- 難易度設定：間違いの数（1個=易、2個以上=難）

### 5. writing（英作文問題）
```json
{
  "id": "qXX-q05",
  "type": "writing",
  "question": "和文英訳の指示（日本語文）",
  "answer": "正解英文",
  "hint": "ヒント",
  "explanation": "模範解答と解説"
}
```
- 用途：総合的な文法理解・応用力確認
- 答え形式：標準的な英文（複数案許可可）
- 難易度：基礎＝文型指定、応用＝自由英作文

### 6. rewrite（書き換え問題）
```json
{
  "id": "qXX-q06",
  "type": "rewrite",
  "question": "元の文と指示（例：Iをheに変える）",
  "answer": "書き換え後の文",
  "hint": "ヒント",
  "explanation": "解説"
}
```
- 用途：文法規則の応用、パターン認識
- 難易度：単純な語の置き換え～複雑な文構造変化

## 難易度調整基準

### 難易度1（易：★）- 中学1年前期相当
- 単語：教科書既出の基本単語のみ
- 文法：単一概念の理解
- 出題形式：4択選択、誘導的なヒント付き
- 文の長さ：5語以内
- 例：He likes soccer. の likes を提示して「このいてい方？」

### 難易度2（標準：★★）- 中学1年後期～2年相当
- 単語：教科書に登場する単語＋中1範囲の単語
- 文法：複合概念（複数の文法ルール交差）
- 出題形式：4択選択、ヒント簡潔
- 文の長さ：5～8語程度
- 例：She studies English every day. の studies を変形箇所として示す

### 難易度3（難：★★★）- 中学2～3年相当
- 単語：教科書範囲＋発展単語
- 文法：複数文法の応用、文脈判断
- 出題形式：自由記述、ヒント最小限
- 文の長さ：8語以上
- 例：「私の妹は毎日、2時間勉強します」を英語で（複数の要素を含む）

## 分量基準

### クイックチェック（単元導入直前）
- 問題数：2問
- 時間：2～3分
- 目的：事前学習の把握、興味喚起
- 難易度：1～2

### 基礎ワーク（理解段階）
- 問題数：10問
- 時間：15分
- 構成：
  - 単純変形：3～4問
  - 穴埋め選択：3～4問
  - 並び替え：2～3問
- 難易度：1～2
- 到達目標：ルール理解＆基本適用

### 応用ワーク（応用段階）
- 問題数：5問
- 時間：10分
- 構成：
  - 英作文：2～3問
  - 複合問題：2～3問
- 難易度：2～3
- 到達目標：複合理解、運用能力

### 小テスト（評価段階）
- 問題数：12問
- 時間：15分
- 配点：60点（各問題に異なる配点）
- 構成：
  - 変形問題：4問（4点×4=16点）
  - 選択問題：2問（4点×2=8点）
  - 並び替え：2問（5点×2=10点）
  - 誤り訂正：1問（5点）
  - 英作文：3問（6～9点）
- 難易度：2～3
- 合格ライン：70点（42点以上）

## 横展開単元スペック

### 1. be動詞（b01）

**単元ゴール**
am/is/areの用法を理解し、主語に応じた正しいbe動詞を選べるようになる。

**核心ポイント**
1. be動詞は存在や状態を表す動詞
2. 主語により am/is/are が異なる（I→am, You→are, He/She/It→is, We/They→are）
3. be動詞は否定文と疑問文が特殊な作られ方をする
4. be動詞は過去形（was/were）へも活用する基礎
5. 他動詞との大きな違い（助動詞を作らない）

**よくあるミス**
1. I are ～ （×）→ I am ～（正）：主語のI忘れ
2. He am ～ （×）→ He is ～（正）：He/She/Itの活用ミス
3. You is ～ （×）→ You are ～（正）：You=複数扱い（中学範囲）

**問題タイプと配分**
- choice（4択主語判定）：40%
- transform（主語別be動詞選択）：25%
- sort（be動詞文の語順）：20%
- writing（be動詞を使った英作文）：15%

**難易度**
★★（標準）

**実装優先順位**
第2位（三単現のsの直後に学習）

**三単現との連携ポイント**
- be動詞は三単現のsルールを適用しない（is≠bes）
- be動詞と一般動詞は現在形でも活用が異なる
- 「Heの後ろ」は is と plays で異なることを強調（概念統一）
- 否定文・疑問文は be動詞 → 一般動詞の順で学ぶ

---

### 2. 一般動詞（g01）

**単元ゴール**
be動詞以外の動詞（一般動詞）の基本用法を理解し、三単現のsを正確に適用できるようになる。

**核心ポイント**
1. 一般動詞は動作や習慣を表す
2. 現在形でも三単現のsを適用（He plays）
3. 一般動詞の否定文・疑問文は do/does を使う
4. be動詞との大きな違い（否定・疑問の作り方）
5. 過去形は -ed の規則活用と不規則活用に分かれる

**よくあるミス**
1. He like soccer （×）→ He likes soccer（正）：sの付け忘れ
2. Does he like soccer? で like → likes にしてしまう（過活用）
3. He no likes ～（×）→ He doesn't like ～（正）：doを使わない否定

**問題タイプと配分**
- transform（三単現s適用）：30%
- choice（do/doesの選択含む）：25%
- sort（一般動詞の疑問文・否定文）：20%
- writing（短い英作文）：15%
- mistake（よくあるミス訂正）：10%

**難易度**
★★（標準）

**実装優先順位**
第3位（be動詞の直後）

**三単現との連携ポイント**
- 実は「一般動詞 = 三単現のs適用対象」が正体
- be動詞と一般動詞は並列に学ぶのではなく「動詞の分類」として整理
- 否定・疑問で do/does が必須であることが、be動詞との最大の違い

---

### 3. 現在進行形（p01）

**単元ゴール**
be動詞 + -ing 形で「今、～している」という継続動作を表せるようになる。

**核心ポイント**
1. 現在進行形 = be + -ing（現在の動作を表す）
2. -ing の付け方ルール（sより複雑：e削除、子音重複など）
3. be動詞の活用を含む（I am, You are, He is...）
4. 時間表現（now, right now, これらは進行形の目印）
5. 一般動詞との「時制の違い」を理解

**よくあるミス**
1. He is play soccer（×）→ He is playing soccer（正）：-ing 忘れ
2. He is playing a soccer（×）→ He is playing soccer（正）：aの付け忘れ
3. make → makeing（×）→ making（正）：e削除ルール

**問題タイプと配分**
- transform（動詞→-ing形）：35%
- choice（be+ing の選択）：25%
- sort（進行形文の語順）：20%
- writing（進行形の英作文）：15%
- mistake（-ing形のミス訂正）：5%

**難易度**
★★（標準）

**実装優先順位**
第4位（be動詞と三単現の定着後）

**三単現との連携ポイント**
- 現在形（He plays）と進行形（He is playing）の時制による使い分け
- be動詞の活用が必須（be動詞の学習が前提）
- -ing は三単現のs と異なるルール（e削除など）→ ルールが違うことを強調

---

### 4. 過去形（p02）

**単元ゴール**
動詞の過去形（規則活用 -ed、不規則活用）を理解し、過去の動作・状態を表せるようになる。

**核心ポイント**
1. 過去形は -ed（規則活用）か不規則活用で表す
2. -ed の付け方ルール（play→played, study→studied）
3. 重要な不規則活用（go→went, have→had, be→was/were）
4. 時間表現（yesterday, last～, ago など）
5. 肯定・否定・疑問の形は現在形と同じ構造

**よくあるミス**
1. He goed（×）→ He went（正）：不規則活用の誤り
2. He studied soccer（×）→ He played soccer（正）：適切な動詞選択
3. study → studyed（×）→ studied（正）：y→ied ルール

**問題タイプと配分**
- transform（規則活用 -ed、不規則活用）：35%
- choice（過去形の選択、特に不規則活用）：25%
- sort（過去形文の語順）：20%
- writing（過去形の英作文）：15%
- mistake（不規則活用や -ed ルールの誤り訂正）：5%

**難易度**
★★★（やや難）

**実装優先順位**
第5位（中1終盤～中2入口）

**三単現との連携ポイント**
- 過去形 -ed の付け方は、三単現のs ルール（study→studies, study→studied）と似ている
- 不規則活用は「ルールより丸暗記」が必須（三単現にはこれがない）
- 過去形の否定・疑問は did + 原形で、did が主役（三単現の does と同じ構造）

### 5. can（c01）

**単元ゴール**
助動詞 can の用法を理解し、能力・可能性を表す英文が作成できるようになる。

**核心ポイント**
1. can は「～できる」（能力・可能性を表す）
2. can の後ろは原形動詞（can plays は不可、can play）
3. can は時制がない（現在・過去ともに can）→ 過去は could
4. 否定形 cannot（can't）、疑問形は Can ～?
5. can の肯定・否定・疑問は自動詞化する（do/does は不要）

**よくあるミス**
1. He can plays soccer（×）→ He can play soccer（正）：can の後ろは原形
2. Can you play soccer? の答え Yes, I can.（do は不要）
3. He can't plays（×）→ He can't play（正）：原形の忘却

**問題タイプと配分**
- choice（can の用法判定）：30%
- transform（肯定・否定・疑問への変形）：25%
- sort（can を含む文の語順）：20%
- writing（can を使った英作文）：15%
- mistake（can の後ろの原形忘れなど）：10%

**難易度**
★★（標準）

**実装優先順位**
第6位（中1の後半）

**三単現との連携ポイント**
- 三単現では He plays（sを付ける）だが、can では He can play（原形）
- can の後ろの動詞は「時制を持たない原形」→ 三単現との対比が重要
- 否定・疑問では「助動詞を主役に」という、一般動詞の do/does の概念と同じ

### 6. 代名詞（p03）

**単元ゴール**
人称代名詞（主格・目的格）と所有代名詞を区別でき、文法的に正しく使い分けられるようになる。

**核心ポイント**
1. 代名詞は「名詞の代わり」→ 文中での位置で主格・目的格が決まる
2. 主格（主語）：I, you, he, she, it, we, they
3. 目的格（目的語・補語）：me, you, him, her, it, us, them
4. 所有格：my, your, his, her, its, our, their
5. 所有代名詞（mine, yours, his, hers, ours, theirs）

**よくあるミス**
1. Me like soccer（×）→ I like soccer（正）：主格と目的格の誤り
2. Him plays soccer（×）→ He plays soccer（正）：目的格を主語に
3. He book（×）→ His book（正）：所有格の忘却

**問題タイプと配分**
- choice（主格・目的格の選択）：35%
- sort（代名詞を含む文の語順）：25%
- transform（名詞→代名詞への置き換え）：20%
- writing（代名詞を正確に使った英作文）：15%
- mistake（主格と目的格の混同）：5%

**難易度**
★（易）～★★（標準）

**実装優先順位**
第7位（中1全般で学習）

**三単現との連携ポイント**
- he/she/it は三単現の判定に重要（I play vs. He plays）
- 代名詞の主格を正確に理解することで、三単現の「どの主語か」の判定が容易に
- 所有格（his）と所有代名詞（his）の区別も重要

### 7. 疑問文（q01）

**単元ゴール**
Yes/No 疑問文と疑問詞疑問文の基本構造を理解し、適切な疑問文を作成・応答できるようになる。

**核心ポイント**
1. Yes/No 疑問文の基本：助動詞/be動詞 + 主語 + 動詞...?
2. be動詞の疑問文：Is he ～? → Yes, he is. / No, he isn't.
3. 一般動詞の疑問文：Does he ～? → Yes, he does. / No, he doesn't.
4. 疑問詞の基本：What, Where, Who, When, Why, How
5. 疑問詞疑問文：疑問詞 + 疑問文の形

**よくあるミス**
1. He like soccer? （×）→ Does he like soccer?（正）：助動詞忘れ
2. Does he likes soccer?（×）→ Does he like soccer?（正）：does の後ろは原形
3. Where he lives?（×）→ Where does he live?（正）：語順誤り

**問題タイプと配分**
- transform（疑問文への変形）：30%
- choice（適切な疑問詞選択）：25%
- sort（疑問文の語順）：25%
- writing（疑問文と応答を含む英作文）：15%
- mistake（疑問文の語順や活用の誤り）：5%

**難易度**
★★（標準）

**実装優先順位**
第8位（中1後期～中2）

**三単現との連携ポイント**
- 疑問文で「Does he play?」となるのは、肯定文の「He plays」の三単現が前提
- Does が主役になるため、動詞は原形に戻る（does plays は不可）
- 疑問詞を使う場合も、助動詞の後ろは原形（Where does he play?）

### 8. 否定文（n01）

**単元ゴール**
be動詞と一般動詞の否定文を区別でき、正確な否定表現ができるようになる。

**核心ポイント**
1. be動詞の否定：be + not（I am not, He is not / isn't）
2. 一般動詞の否定：do/does + not + 原形動詞
3. 短形：isn't, aren't, doesn't, don't
4. nobody, nothing などの否定代名詞
5. 肯定→否定の変形は「どの助動詞か」で決まる

**よくあるミス**
1. He not plays soccer（×）→ He doesn't play soccer（正）：do/does を使わない
2. He no like soccer（×）→ He doesn't like soccer（正）：正しい否定構造
3. He is not play soccer（×）→ He is not playing soccer（正）：be の後ろは ing

**問題タイプと配分**
- transform（肯定→否定への変形）：35%
- choice（正しい否定形の選択）：25%
- sort（否定文の語順）：20%
- writing（否定を含む英作文）：15%
- mistake（否定文の構造的誤り）：5%

**難易度**
★★（標準）

**実装優先順位**
第8位同等（疑問文と同時期、相互補完）

**三単現との連携ポイント**
- 否定文で「doesn't play」となるのは、肯定文の「plays」の三単現が前提
- does が出現するため、動詞は原形に戻る（doesn't plays は不可）
- 三単現の理解がなければ「なぜ play に戻るのか」が理解困難

## 全単元共通の教材フォーマット定義

### JSONの必須フィールド

#### 1. unit セクション（必須）
```json
"unit": {
  "id": "sXX",              // 単元ID（s=三単現, b=be動詞, g=一般動詞など）
  "title": "～",            // 日本語単元名
  "grade": 1,               // 学年（1/2/3）
  "difficulty": 1,          // 難易度（1/2/3）
  "totalQuestions": 29,     // 総問題数（計算値）
  "estimatedMinutes": 20,   // 推定学習時間
  "goal": "～",             // 単元ゴール（1文）
  "description": "～"       // 単元説明（簡潔）
}
```

#### 2. explanation セクション（必須）
- 最低3つの subsection（理論、ルール、よくあるミス）
- 各セクションに keyPoints と examples を含む
- examples は en（英語）と ja（日本語）のペア

#### 3. quickCheck セクション（必須）
- 2問固定
- 単元導入直前の実力把握
- choice 型を推奨

#### 4. exercises セクション（必須）
- basic：10問（難易度1～2）
- advanced：5問（難易度2～3）

#### 5. miniTest セクション（必須）
- 12問固定
- 60点満点（passingScore: 70 = 42点以上）
- 配点：4点/5点/6点/9点を使用

#### 6. teacherNote セクション（必須）
- 指導のポイント
- 定期テスト頻出ポイント
- 導入の流れ
- 授業時間の目安

#### 7. faq セクション（必須）
- 最低4つの質問と回答

#### 8. metadata セクション（必須）
```json
"metadata": {
  "version": "1.0",
  "createdDate": "YYYY-MM-DD",
  "author": "中学校英語サポート＋開発チーム",
  "language": "ja",
  "encoding": "utf-8"
}
```

## 表記統一ガイド

### 英語
- 文の最後に . をつける
- 人名は大文字（Tom, Yuki）
- 動詞は小文字（plays, studies）

### 日本語
- 句点は「。」（。を使う）
- 「～ます」調ではなく「～る」調
- 括弧は「（）」（全角）
- 単位は「～分」「～点」

### コード
- JSON は UTF-8 エンコーディング
- インデント 2 スペース
- キーは英語のキャメルケース
